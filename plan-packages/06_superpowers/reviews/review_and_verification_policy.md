# Human Verification Policy

版本：v0.4  
日期：2026-05-22  
状态：draft

## Purpose

本项目用于比赛 Demo，目标是视觉效果和实际运行效果优先。代码实现由 Qoder 完成，验证由人工负责人按任务包中的验证条目实机运行完成。

本阶段不要求将每个 Job 返回给 Codex 做代码审查，也不要求执行高强度工程质量 review。

## Verification Gates

每个实现 Job 完成后必须经过人工验证：

1. Qoder 输出本 Job 的实现说明。
2. Qoder 输出本 Job 的验证条目。
3. 人工负责人按条目在本机运行。
4. 人工记录结果：通过、失败、可接受瑕疵、需返工。
5. 若失败，交给 Qoder 修复。

## Verification Gate

完成声明前必须提供：

- 人工执行的验证条目。
- 本机运行结果。
- 关键截图或页面现象。
- 已知问题和是否影响比赛演示。

未运行验证时，只能说“未验证”，不能说“完成”。

## Demo Acceptance Bias

比赛阶段优先级：

1. 页面能打开。
2. 样例日志能跑通。
3. PID 生命周期、动态 Tag、时间线、AI 报告可见。
4. DeepSeek 接入可用或 mock 可稳定兜底。
5. 答辩叙事闭环清楚。

以下问题可记录但不阻塞比赛 Demo：

- 高压大日志性能不充分。
- 边缘日志格式覆盖不全。
- UI 细节不够精致。
- 单元测试覆盖不完整。

以下问题必须阻塞：

- 页面无法启动。
- 样例链路无法跑通。
- API key 暴露到前端或仓库。
- DeepSeek/mock 都不能生成报告。
- 报告把 AI 推测写成确定事实。
