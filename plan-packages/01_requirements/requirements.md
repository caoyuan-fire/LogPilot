# Requirements：LogPilot

版本：v0.4  
日期：2026-05-22  
状态：draft

## 1. 项目简介

LogPilot 是一个面向 Android/座舱系统缺陷排查的 AI 日志裁剪与缺陷分诊助手。它以时间窗口和多个包名为入口，追踪 PID 生命周期，动态发现本次复现周期内的 Log Tag，提取高价值日志证据，生成事件时间线，并输出 AI 分诊报告。

## 2. 用户

- 测试工程师。
- Android/座舱应用研发。
- HealthMonitor 研发和运维人员。
- 系统集成工程师。
- 项目交付和质量负责人。

## 3. 使用场景

- 测试复现 Bug 后上传大量日志，需要快速提取有效信息。
- HealthMonitor 发现异常后自动上传日志并建 Jira 单。
- 研发接单前需要看到目标进程、异常 Tag 和关键事件线。
- 多个模块互相影响，需要输入多个包名进行跨进程分析。

## 4. 功能需求

R1：日志导入  
支持导入 logcat 或类似 Android/座舱日志文本。

R2：复现窗口  
支持输入问题发生时间点或复现起止时间。只输入时间点时，默认使用前后 5 分钟。

R3：多包名输入  
支持一个或多个包名/进程名。模块选择由人工决定，系统不凭空猜测。

R4：PID 生命周期追踪  
识别每个包名在窗口内的启动、死亡、重启、多 PID 片段。

R5：目标 PID 日志裁剪  
基于 PID 而不是静态 Tag 提取目标日志。

R6：动态 Tag 发现  
统计目标 PID 在复现周期内出现的所有 Tag、频次、等级分布、首次/末次出现时间。

R7：系统关键事件补充  
补充 ANR、crash、kill、timeout、FATAL、Watchdog、Input dispatching timed out 等系统事件。

R8：事件时间线  
生成跨包名、跨 PID、按时间排序的关键事件时间线。

R9：AI 分诊报告  
基于结构化摘要生成报告，包含证据、可疑点、排查建议、回归建议和 Jira 评论版。

R10：HM/Jira 闭环  
支持用模拟或真实接口展示 HM 发现异常、上传日志、建单、AI 分诊、回填 Jira 的流程。

R11：AI 接入与 Web 本地开发  
系统应支持本地 Web Demo 形态，前端负责输入和展示，后端负责日志处理与 AI 调用代理。AI 接入必须支持 mock 和 DeepSeek API 两类模式；默认开发模式为 mock，无 API key 也能跑通完整流程；比赛真实 AI 效果默认使用 DeepSeek。真实 API key/token 只能存放在后端 `.env.local`，不得进入前端或代码仓库。

## 5. 非功能需求

- AI token 成本可控。
- 报告结论可追溯到日志证据。
- 支持脱敏。
- 异常输入有明确错误提示。
- Demo 稳定优先于功能完整。
- Web Demo 应提供明确的本地启动、调试和验证流程。
- 真实模型接入应默认预置 DeepSeek provider，可配置 base URL、model name、timeout 和 API key/token。

## 6. 验收标准

- 输入样例日志、时间窗口、两个包名，能输出 PID 生命周期。
- 能输出动态 Tag 统计。
- 能输出关键事件时间线。
- 能生成证据驱动的 AI 分诊报告。
- 能解释与通用 Agent 客户端的差异。
- 能讲清楚 HM/Jira 闭环的业务价值。
- `AI_PROVIDER=mock` 时无需 key 即可运行 Demo。
- 如切换 DeepSeek 真实模型，模型调用必须经由后端代理，前端不得直接持有 API key。
- 每个 Job 必须输出人工验证条目，Qoder 实现后由人工按清单实机运行验证。
